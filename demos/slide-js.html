function slide() {
  var timestamp = Date.now();
  var progress = timestamp - start;
  var x = Math.min(1, progress / SLIDE_TIME) * -width;
  $slider.css('left', x + 'px');

  if (progress < SLIDE_TIME) requestAnimationFrame(slide);
}

Clunky, right?

It's 2013 - stop using JavaScript to animate things! It's almost never a good idea. Beyond the increased CPU load, bad performance, and terrible user experience, it mixes presentation with business logic. JavaScript-based animations are harder to write and maintain, and they don't deliver many of the features available in CSS.

Some developers still use JS-based animation because it's baked into jQuery. If you're developing a mobile solution, check out Zepto.js instead - it follows the jQuery API but implements mobile features like CSS-only animations.
